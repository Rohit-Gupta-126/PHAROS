# PHAROS wire format (Phase 1)

All topics carry UTF-8 JSON, one record per Kafka message, keyed by
`event_id`. Every schema is versioned via a `schema` string; consumers must
reject records whose schema they don't recognize.

Design rule: **producers emit pre-normalization, model-ready-shape data**. The
scorer owns normalization (loads the Phase 0 stats and applies the exact
training transform), so there is a single source of truth and the stream
mirrors "raw data in".

## `events.physics` — `pharos.physics.v1`

One ADC2021 event per message.

| field            | type        | notes                                            |
|------------------|-------------|--------------------------------------------------|
| `schema`         | str         | `"pharos.physics.v1"`                            |
| `event_id`       | str         | `phys-<seq>-<hex8>`, unique                      |
| `producer_ts_ns` | int         | wall-clock ns at emit time                       |
| `features`       | float[57]   | raw `[pT, eta, phi] × 19` slots (MET, 4e, 4μ, 10 jets); class column dropped; **no** log1p/z-score |

## `events.pdm` — `pharos.pdm.v1`

One HVCM pulse per message (~110 KB JSON at target_len=500).

| field            | type            | notes                                        |
|------------------|-----------------|----------------------------------------------|
| `schema`         | str             | `"pharos.pdm.v1"`                            |
| `event_id`       | str             | `pdm-<SYSTEM>-<seq>-<hex8>`                  |
| `producer_ts_ns` | int             | wall-clock ns at emit time                   |
| `system`         | str             | `RFQ` \| `DTL` \| `CCL` \| `SCL`             |
| `shape`          | int[2]          | `[target_len, 14]` (time-major)              |
| `wave`           | float[500][14]  | avg-pooled 4500→500 waveform, **pre-norm**   |
| `ground_truth`   | str             | `Run` \| `Fault` — offline keep-rate accounting only; scorers must not use it |

## `anomalies.scouting` — `pharos.scouting.v1`

Physics events whose Σμ² exceeded the derived threshold.

| field            | type  | notes                                   |
|------------------|-------|-----------------------------------------|
| `schema`         | str   | `"pharos.scouting.v1"`                  |
| `event_id`       | str   | copied from the input event             |
| `score`          | float | Σμ² (AXOL1TL convention)                |
| `threshold`      | float | threshold in force when kept            |
| `producer_ts_ns` | int   | copied from the input event             |
| `scored_ts_ns`   | int   | scorer output time; `scored - producer` is the e2e latency |

## `alerts.pdm` — `pharos.pdm_alert.v1`

Same as scouting plus `system`; `score` is the conv-AE reconstruction MSE.

## Thresholds

`configs/thresholds.json` is generated by `make thresholds`
(`scripts/derive_thresholds.py`): the p99 (configurable) of the Phase 0
background/normal score distribution per detector, so the expected background
keep-rate is `(100 − percentile)/100` by construction.
